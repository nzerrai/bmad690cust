- (decision) Source: ux-design-specification.md (Nouredine, 26 mars 2026). Product: GenData — internal test dataset generation platform for 1000+ banking developers/QA. Transforms CSV into realistic configurable datasets via 112 banking typologies with deterministic seed reproduction.
- (decision) Form-factor: Web app only. React TypeScript, desktop-first (1920x1080 target), responsive down to tablet (1024px). Mouse/keyboard primary. No offline mode. Mobile <768px out of scope for MVP.
- (decision) UI System: MUI v6 (layout, forms, tables, navigation, TreeView), @mui/x-data-grid (dataset preview virtualization 1M rows), @mui/x-tree-view (sidebar Tribu→Projet→Sous-projet), Monaco Editor (inline SQL editor for DDL import), React Flow (FK dependency graph), react-jsonschema-form (auto-generated typology forms from configSchema), Recharts (usage metrics charts).
- (decision) Navigation model: Hierarchical Tribu→Projet→Sous-projet→[Dataset|DDL]. N1 label configurable (Tribu/Département/Équipe/Domaine/BU) set at workspace creation. Sidebar 260px fixed, never collapsible on desktop. Deep-linkable URLs per resource.
- (decision) Design direction: Dense & Trustworthy — bureautique, bancaire, professionnelle. Confiance avant esthétique. Pas de gradients, pas d'espacement marketing, pas de couleurs pastel, pas de hamburger sur desktop.
- (decision) Colors: primary #1565c0 (Blue 800), primary-hover #1976d2, primary-pale #e3f2fd, surface #ffffff, background #f5f5f5, border #e0e0e0, border-hover #90caf9, success #2e7d32, success-pale #e8f5e9, warning #f57c00, warning-pale #fff3e0, error #d32f2f, text-primary #1a1a2e, text-secondary #757575, text-disabled #9e9e9e (decoratif only), seed-bg #f5f5f5/border #e0e0e0, sql-dark #1e272c, sql-keyword #80cbc4. WCAG AA: primary sur white 5.9:1, success 5.4:1, text-secondary 4.6:1.
- (decision) Typography: Inter (body, labels), Roboto Mono (data, seeds, SQL code). Page Title 28px/800, Section Title 22px/700, Sub-section 16px/700, Body 13px/400, Body Strong 13px/600, Caption 12px/400, Label Uppercase 11px/700 (letter-spacing 0.7px), Monospace 13px/400.
- (decision) Spacing: base 4px grid. space-xs 4px, space-sm 8px, space-md 12px, space-lg 16px, space-xl 20px, space-2xl 24px, space-3xl 32px. Desktop layout: topbar 48px fixed z-100 bg #1565c0, sidebar 260px fixed bg #ffffff overflow-y auto, content area flex:1 padding 24px max-width none. Principles: density (12-16px card padding), permanent sidebar, scrollable content, grid auto-fill cards repeat(auto-fill, minmax(280px, 1fr)).
- (decision) Accessibility: WCAG 2.1 AA target. All functional text >=4.5:1 contrast. Focus ring #1565c0 2px offset 2px on all interactive elements (never outline:none without alternative). Keyboard nav logical tab order in sidebar, config table, drawers — no traps except modals. ARIA: aria-label on confidence badges, aria-checked on toggles, role=progressbar + aria-valuenow/aria-valuemax on generation progress bar. Touch targets min 32x32px desktop, 44x44px tablet. Respects prefers-reduced-motion. lang=fr.
- (decision) Emotional targets: Discovery='Wow, c'est simple' (surprise positive), Generation success='C'est exactement ce qu'il me faut' (satisfaction), Navigation='Je suis chez moi' (appartenance), DDL/Power features='Cet outil me comprend' (empowerment), Seed reproduction='Je maîtrise' (contrôle). Micro-emotions à favoriser: confiance (IBAN checkmark visible, preview avant bulk), sentiment progression (wizard étapes numérotées), délice (auto-detect >95% confidence animation subtile), appartenance (avatar tribu, compteur membres), fierté (partager dataset bien structuré). À éviter: doute, confusion, frustration, isolement, honte données fausses.
- (decision) Button hierarchy: Primary (bg #1565c0 white text, border-radius 6px) — unique action per view. Success (bg #2e7d32 white text) — irreversible bulk generation. Secondary (bg white, border #1565c0, blue text) — reversible actions like Preview. Ghost/Icon (transparent bg, icon only) — ⚙️ + ··· Danger (#f44336 transparent bg) — destructive dropdowns. Rule: one primary per page.
- (decision) Feedback patterns: Success=green Alert + Snackbar (#e8f5e9/#2e7d32, auto-dismiss 5s). Info/guidance=blue Alert (#e3f2fd/#1565c0). Attention required=orange Alert (#fff3e0/#f57c00). Error recoverable=red Alert (#fce4ec/#d32f2f). Progression async=GenerationProgressCard blue animated bar. Rules: congratulate before inform, never show raw stack trace, snackbar 5s success / manual dismiss error.
- (decision) Form patterns: labels above input (never placeholder-only). Inline validation on blur with red HelperText. Required fields: red asterisk + 'Champs requis' note at bottom. Defaults: seed=random pre-filled, lignes=1000, format=JSON. Numeric inputs: no up/down arrows, direct entry only. Contextual widths for seed/lignes.
- (decision) Navigation patterns: sidebar always visible on desktop, active item bg #e3f2fd text #1565c0. Breadcrumb on all views, last element non-clickable. Tabs blue underline 2px active, content filters immediately. Deep-links synced with navigation — copy URL = share exact context. No explicit back button (breadcrumb + sidebar suffice).
- (decision) Empty/loading states: empty list=illustrative icon + contextual message + CTA [+ Créer le premier Dataset]. Initial load=skeleton loaders (MUI Skeleton), no global spinner. Inline loading (auto-detect)=16px CircularProgress in Confidence column. Load error=inline message + Retry button, not full error page.
- (decision) Search/filtering: Typology Browser=real-time search (debounce 300ms) + category filter chips. Dataset list=full-text search within Sub-project context only (never cross-workspace). Reset filters button visible only when a filter is active.
- (decision) Modal/overlay patterns: Column config (typo + Test Lab)=Drawer 400px slide-in right, table visible left. Destructive confirmation=Centered Dialog with explicit title + danger button + cancel. N1 label config=Inline panel in sidebar header (not a modal). Help/documentation=enriched Tooltip on hover over confidence badges and technical icons. Rule: Dialogs reserved for destructive confirmations only; everything else = Drawer or inline panels.
- (decision) Custom components: C-01 ConfidenceBadge (high>=80% green #2e7d32, medium 50-79% orange #f57c00, low<50% red #d32f2f, aria-label='Confiance {v}%', role=status). C-02 SeedBadge (default + copied state with checkmark animation 1.5s, clickable copy clipboard, aria-label='Seed de reproductibilité: {v}. Cliquer pour copier'). C-03 TypologyChip (validated blue #e3f2fd, warning orange #fff3e0, manual purple pale; click opens filtered drawer). C-04 ColumnConfigRow (# · col-name · TypologyChip · ConfidenceBadge · toggle Nullable · toggle Unique · btn ⚙️; states: normal white bg, attention #fff8e1+orange highlight, locked FK DDL).
- (decision) Custom components continued: C-05 FKGraphCanvas (React Flow, node states: parent green no incoming FK, child blue, selected border highlight; drag reposition, hover FK tooltip TABLE_A.col→TABLE_B.col). C-06 GenerationProgressCard (states: pending/running with progress bar + columns in progress/completed/failed, role=progressbar aria-valuenow aria-valuemax). C-07 LabelSettingsPanel (closed/open state, revealed by ⚙ click in sidebar header; anatomy: predefined chips Tribu/Département/etc. + free input + OK button).
- (decision) Key flows: Flow1 Yann=CSV→Generation nominal (navigate sidebar → +New Dataset → drag-drop CSV → auto-detect <2s → config table green/orange lines → fix orange via drawer → generation config → preview 5 lines → generate N rows → success screen with download/copy seed/share link). Flow2 Fatima=Manual correction + Test Lab (CSV non-standard headers → orange badge → drawer Typology Browser search → select typo → open Test Lab inline → configure range → live examples <500ms → assign → green line 98%). Flow3 DDL=SQL import → FK graph analysis <3s → cycle detection if any → FK graph React Flow → config generation per table + seed + format → assign typologies → generate schema.
- (decision) Flow4 Omar=Onboarding zero-friction (arrive first time → sidebar shows existing spaces → click team's KYC Department → see team projects → click colleague's dataset → page with all configs → click Regenerate with seed:42 → async job <10s for 1000 rows → identical dataset → optional exploration via Typology Browser + Test Lab). Cross-cutting patterns: contextual side drawer (typo correction, Test Lab, column config), color-guided alerts (green=valid, orange=action required, red=blocking), permanent seed visible, async progression visible, non-destructive undo, stable deep-link URLs.
- (decision) Responsive strategy: Desktop 1440-1920px primary (full screen, fixed sidebar 260px, no max-width content). Desktop 1280-1440px identical fluid layout. Tablet >=1024px same layout with touch targets 44px min. Tablet <1024px collapsible sidebar overlay hamburger — consultation only, no column config. Mobile <768px out of scope MVP — show 'tool not available on mobile' page. Desktop-first approach with @media (max-width: 1023px) for tablet adaptations.
- (decision) Dashboard workspace (/workspaces/:id): Page header with N1 label + workspace name + member count + created date. 4 KPI cards (Projects count, Datasets total, Rows generated, Last activity) — value 28px/800, delta 12px/400 green #2e7d32 for positive. Projects section: grid repeat(auto-fill, minmax(260px,1fr)) max 6 cards sorted by recent activity, each card shows subprojects count + datasets count + last active time + [Open →] CTA + [...] context menu (Rename, Archive). Activity feed: chronological list grouped by day max 10 entries with event icons (✅ generated, 📤 exported, 🗄️ DDL generated, 📁 project created, 📂 subproject created).
